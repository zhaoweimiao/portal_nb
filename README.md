# Portal — 团队内部门户与知识库系统

团队内部使用的门户(Portal)与知识库系统的需求、调研与实施方案。知识库采用**文件库模式**,基于开源项目 **moredoc(Go)** 二次开发;门户作为轻量外壳后续补充。

## 项目背景

- 团队规模:约 100~200 人
- 部署:单台服务器,内网为主,Docker 容器化
- 认证:一期自建账号,二期对接钉钉 OAuth2
- 知识库定位:目录创建、知识分类、文件上传/描述/删除、文件检索、内容检索
- 存储:本地存储起步,预留 S3/MinIO 对象存储(尽量支持)

## 技术选型摘要

- 知识库:moredoc(Go + Vue),二次开发,裁剪其面向 C 端的功能
- 数据库:MySQL
- 全文检索:Elasticsearch(中文)
- 存储:本地 → 预留 MinIO/S3
- 统一认证:moredoc 自带账号 → Casdoor 对接钉钉
- 部署:Docker + Docker Compose,Nginx 反向代理
