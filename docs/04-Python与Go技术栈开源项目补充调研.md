根据需求的进一步明确，本次补充调研的硬性条件为：

## 3. 重点项目深度分析

### 3.1 moredoc 魔豆文库（Go）★ 文件型知识库首选

**项目简介：** moredoc（魔豆文库）是基于 Go 语言开发的类百度文库开源文库系统，是 dochub 文库的重构版本。它专注于"文件上传 → 在线浏览 → 分类管理 → 检索下载"的完整流程。

**核心能力：**
- 支持 Office（全部类型）、PDF、TXT、EPUB、MOBI 等多种文档格式的**在线阅读浏览**（自动转换预览）
- 支持**无限级分类**
- 支持文档**批量上传**、批量转换
- 支持**全文搜索**，v2.4.0 起支持 **Elasticsearch**
- 支持 Docker 部署

**存储后端能力（本项目最大亮点）：**
moredoc 是所有候选项目中对象存储支持最全面的，原生支持：
- 本地存储 Local
- 亚马逊对象存储 AWS（S3 标准）
- 混合云/多云对象存储 MinIO
- 阿里云 OSS、腾讯云 COS、百度云 BOS、华为云 OBS、天翼云 ZOS

通过 `app.toml` 配置文件即可在本地存储和各类对象存储之间切换，**完美满足"S3 + 本地存储"的需求**。

**与本项目需求匹配度：★★★★★（针对文件型知识库）**

你的原始需求中知识库部分明确列出了"文件上传、文件描述、文件删除、文件检索、内容检索"——这是一个**以文件为中心**的知识库，moredoc 的定位与之高度吻合。它天然就是为"上传文件 → 描述分类 → 检索下载"设计的。

**局限性：**
- 偏向"文档库/文库"模式，在线**协同编辑**能力弱（不是 Notion/语雀那种在线写作）
- 自带 VIP、支付（支付宝/微信）等 toC 文库功能，团队内网使用时需要裁剪关闭
- 没有门户功能，需自研或搭配 CMS

### 3.2 MrDoc 觅思文档（Python/Django）★ Wiki型知识库首选

**项目简介：** MrDoc（觅思文档）是基于 Python + Django 开发的开源在线文档系统，被称为"可私有部署的语雀"和"可在线编辑文档的 GitBook"。当前版本 v0.9.9，基于 Python 3.9+ 和 Django 4.2。

**核心能力：**
- 支持 **Markdown 和富文本**两种编辑方式，提供 3 种编辑器
- 支持插入数学公式、流程图、序列图、脑图、Echarts 图表、时间线
- 支持音视频链接、图片附件、文档模板
- **文集（知识空间）+ 文档**的层级组织
- 4 种权限模式：公开、私密、指定用户可见、访问码可见
- **版本管理**：历史版本对比，便于团队协作
- 支持导出 Markdown 压缩包、EPUB、PDF
- 丰富的客户端生态：浏览器剪藏扩展、桌面客户端、移动客户端、Obsidian 同步插件

**存储后端能力：**
- 数据库支持 SQLite、MySQL、PostgreSQL、Oracle
- 通过 `config/config.ini` 配置存储后端
- 由于基于 Django，可借助 **django-storages** 库实现 S3 兼容对象存储（MinIO/OSS 等）+ 本地存储切换（此方案为 Django 生态通用做法，需在部署时配置验证）

**与本项目需求匹配度：★★★★★（针对 Wiki 型知识库）**

如果团队期望的知识库是"**在线写作 + 结构化文档 + 版本管理**"的 Wiki/语雀模式，MrDoc 是 Python 生态中最成熟的选择。Django 技术栈主流、文档完善、二次开发门槛低。

**局限性：**
- 前端基于 LayUI + JQuery，技术栈相对传统，前端二次开发体验不如现代框架
- 实时协同编辑能力有限
- S3 存储需通过 django-storages 配置，非开箱即用的一键配置
- 同样没有门户功能

### 3.3 MinDoc（Go/Beego）— 团队技术文档场景

**项目简介：** MinDoc 是 Go 生态中最经典的团队文档管理系统，基于 Golang + Beego 框架，前身为 SmartWiki。

**核心能力：**
- Markdown 和富文本两种编辑模式
- 项目（文档集）管理、用户权限控制
- 文档历史版本恢复
- 附件管理
- 多格式导出（PDF、Word、EPUB、Markdown）
- 单二进制部署，运维简单

**存储后端能力：**
- 默认本地存储
- 附件支持配置阿里云 OSS 等（对 S3 标准的支持不如 moredoc 全面，需评估）

**与本项目需求匹配度：★★★★☆**

非常适合做 IT 团队的 API 文档、数据库字典、流程手册。部署极简（单二进制）。但其对象存储支持不如 moredoc 全面，且界面较为传统。

---

### 3.4 Documize（Go + EmberJS）— 协作文档场景

**项目简介：** Documize 是基于 Go + EmberJS 的 Confluence 开源替代品，编译为单一二进制文件。

**核心能力：**
- Spaces / Labels / Categories 内容组织
- 多种内容段类型：富文本、代码、Markdown、Jira、Trello 等
- 原生支持 LDAP、AD、Keycloak/CAS SSO
- 全文搜索
- 内容消费分析

**存储后端能力（关键短板）：**
- 数据存储于关系型数据库（PostgreSQL / MySQL / MS SQL Server）
- **不支持 S3 对象存储作为文件存储后端**

**与本项目需求匹配度：★★★☆☆**

认证和协作能力强，但**不满足 S3 存储的硬性要求**，因此不作为本项目首选。

---

### 3.5 Wagtail（Python/Django）— 门户补充方案

**项目简介：** Wagtail 是基于 Django 的企业级开源 CMS，被众多大型网站采用。

**适用定位：** 作为**门户 Portal** 的内容管理后端，管理公告、新闻、页面等结构化内容。

**存储能力：** 基于 Django，原生通过 django-storages 支持 S3 + 本地存储。

**说明：** Wagtail 更偏 CMS/门户，知识库能力需定制。可作为"门户层"与上述知识库项目（moredoc/MrDoc）组合使用。

## 4. 推荐方案

### 4.1 决策树

```
知识库的核心使用方式是？
│
├── 以"上传文件 + 下载 + 检索"为主（文件库模式）
│   └── 首选：moredoc 魔豆文库（Go）
│       理由：对象存储支持最全面，天然契合文件型知识库，
│             全文检索+ES，在线预览Office/PDF
│
└── 以"在线写作 + 结构化文档 + 版本管理"为主（Wiki模式）
    └── 首选：MrDoc 觅思文档（Python/Django）
        理由：Wiki式知识库最成熟，Django生态便于S3集成和二次开发
```

### 4.2 综合推荐

结合你的需求描述——知识库要支持"目录创建、知识分类、文件上传、文件描述、文件删除、文件检索、内容检索"，这是一个**偏文件管理 + 检索**的知识库，因此：

**首选推荐：moredoc 魔豆文库（Go）**

推荐理由：

1. **存储需求完美匹配。** moredoc 是所有候选中唯一原生、全面支持 S3/MinIO/OSS + 本地存储的项目，配置文件一键切换，完全满足你的硬性要求。

2. **功能与需求高度对齐。** 你列出的知识库功能（无限级分类、文件上传、文件描述、检索、内容检索）正是 moredoc 的核心能力，几乎不需要额外开发。

3. **在线预览能力强。** 自动转换并在线预览 Office、PDF、EPUB 等格式，用户无需下载即可查看。

4. **全文检索成熟。** 支持 Elasticsearch，中文检索效果有保障。

5. **Go 单服务，资源占用低。** 适合你单台 CPU 服务器的部署环境。

**需要的改造工作：**
- 裁剪/关闭 toC 文库功能（VIP、支付、注册开放等），改造为内网团队使用
- 对接统一认证（可保留其手机号登录或改造为对接 Casdoor/钉钉）
- 门户部分：自研轻量前端，或搭配 Wagtail 作为门户 CMS

**备选推荐：MrDoc 觅思文档（Python）**

如果团队更看重"在线写作 + Wiki 协作"的体验，且偏好 Python 技术栈，MrDoc 是更好的选择。它提供完整的 Wiki 式知识库、版本管理和丰富的客户端生态，S3 存储可通过 Django django-storages 配置实现。

### 4.3 门户的处理

由于本次优先保证知识库，门户可采用以下两种方式之一：

1. **轻量自研门户**：用 Go（Gin）或 Python（Django/FastAPI）写一个简单的门户前端，复用知识库的认证和 API，展示公告 + 快捷入口 + 知识库入口。
2. **搭配 Wagtail**：用 Wagtail 作为门户 CMS 管理公告和页面，知识库通过链接/iframe 集成。

无论哪种方式，知识库（moredoc 或 MrDoc）都作为独立核心服务先行落地，门户作为轻量外壳后续补充。

## 5. 技术栈对比小结

| 维度 | moredoc (Go) | MrDoc (Python) |
|------|-------------|----------------|
| 知识库模式 | 文件库（上传/检索/预览） | Wiki（在线写作/版本） |
| S3 对象存储 | ✅ 原生全面 | ✅ django-storages |
| 本地存储 | ✅ | ✅ |
| 全文检索 | ✅ 支持 Elasticsearch | ✅ 内置 |
| 在线预览 | ✅ Office/PDF/EPUB | ⚠️ 主要 Markdown |
| 在线协作编辑 | ⚠️ 弱 | ⚠️ 一般 |
| 部署复杂度 | 低（单服务） | 中（Django） |
| 二次开发 | Go + Vue | Python + LayUI |
| 与需求匹配 | ★★★★★（文件型） | ★★★★★（Wiki型） |
