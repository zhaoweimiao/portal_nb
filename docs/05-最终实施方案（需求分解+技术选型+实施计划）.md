# 团队门户与知识库系统 — 最终实施方案

> 文档版本：v2.0（最终版）  
> 创建日期：2026-06-06  
> 方案基调：知识库采用**文件库模式**，基于 **moredoc 魔豆文库（Go）** 二次开发  
> 说明：本文档为最终落地方案，整合需求分解、技术选型与实施计划三部分。此前基于 Outline(Node.js) 的实施方案（文档03）已被本方案取代。

---

## 一、方案总览

| 项目 | 决策 |
|------|------|
| 知识库模式 | 文件库模式（上传 → 分类 → 描述 → 检索 → 在线预览 → 下载） |
| 知识库选型 | moredoc 魔豆文库（Go + Vue），二次开发 |
| 门户选型 | 轻量自研外壳，知识库先行落地 |
| 存储后端 | 优先本地存储起步，预留 S3/MinIO 对象存储（moredoc 原生支持，按需切换） |
| 认证方式 | 一期自建账号；二期对接钉钉 OAuth2 |
| 部署 | 单台 CPU 服务器 + Docker Compose |
| 团队规模 | 100~200 人 |

## 二、需求分解

将系统拆解为"门户"与"知识库"两大域，知识库为核心、优先落地。下面按"功能模块 → 功能点 → 在 moredoc 中的实现方式"三层分解，明确哪些是 moredoc 现成能力、哪些需要二次开发。

### 2.1 知识库域（核心，优先级 P0）

#### 模块 A：目录与分类管理

| 功能点 | 说明 | moredoc 现状 | 工作量 |
|--------|------|-------------|--------|
| A1 多级目录/分类创建 | 支持无限级分类树 | ✅ 原生支持 | 配置 |
| A2 分类编辑/排序/删除 | 后台管理分类 | ✅ 原生支持 | 配置 |
| A3 知识分类标签 | 文档打标签便于聚合 | ✅ 支持标签 | 配置 |

#### 模块 B：文件管理

| 功能点 | 说明 | moredoc 现状 | 工作量 |
|--------|------|-------------|--------|
| B1 文件上传 | 单个/批量上传 | ✅ 支持批量上传 | 配置 |
| B2 多格式支持 | Office全类型/PDF/TXT/EPUB/MOBI | ✅ 原生支持 | 配置 |
| B3 文件描述/摘要 | 标题、描述、分类、标签等元信息 | ✅ 支持 | 配置 |
| B4 文件在线预览 | 文档自动转换在线浏览 | ✅ 核心能力 | 配置 |
| B5 文件下载 | 单文件下载 | ✅ 支持 | 配置 |
| B6 文件删除 | 删除文档 | ✅ 支持 | 配置 |
| B7 回收站 | 删除保留期与恢复 | ⚠️ 需确认/增强 | 小量开发 |
| B8 版本管理 | 文件更新版本记录 | ⚠️ 弱，按需增强 | 中量开发 |

#### 模块 C：检索

| 功能点 | 说明 | moredoc 现状 | 工作量 |
|--------|------|-------------|--------|
| C1 文件检索（元数据） | 按文件名/分类/标签/时间筛选 | ✅ 支持 | 配置 |
| C2 内容检索（全文） | 文档正文全文检索 | ✅ 支持，可接 Elasticsearch | 配置 |
| C3 中文分词检索 | 中文搜索准确性 | ✅ ES 方案保障 | 配置 |

#### 模块 D：存储后端

| 功能点 | 说明 | moredoc 现状 | 工作量 |
|--------|------|-------------|--------|
| D1 本地磁盘存储 | 默认存储方式 | ✅ 原生支持 | 配置 |
| D2 S3/MinIO 对象存储 | 可切换至对象存储 | ✅ 原生支持多云 | 配置（按需启用） |

#### 模块 E：权限（知识库内）

| 功能点 | 说明 | moredoc 现状 | 工作量 |
|--------|------|-------------|--------|
| E1 分类/文档访问权限 | 公开/受限可见 | ✅ 支持 | 配置 |
| E2 角色权限 | 管理员/普通用户等 | ✅ 支持 | 配置 |

### 2.2 门户域（次要，优先级 P1）

由于知识库优先，门户先做轻量外壳，复用知识库的认证体系。

| 功能点 | 说明 | 实现方式 | 工作量 |
|--------|------|----------|--------|
| F1 统一首页 | 公告 + 快捷入口 + 知识库入口 | 自研轻量前端 | 中量开发 |
| F2 公告/通知管理 | 发布、置顶、撤回 | 自研后端 CRUD | 中量开发 |
| F3 快捷入口 | 常用系统链接配置 | 自研后端 CRUD | 小量开发 |
| F4 信息编辑后台 | 后台编辑、发布 | 自研管理界面 | 中量开发 |
| F5 知识库无缝跳转 | 全局可访问知识库 | 链接/嵌入 | 配置 |

### 2.3 公共域（认证、运维）

| 功能点 | 说明 | 实现方式 | 优先级 |
|--------|------|----------|--------|
| G1 自建账号登录 | 用户名/密码 | moredoc 自带 + 改造 | P0 |
| G2 钉钉 OAuth2 登录 | 从钉钉跳转 | 二期，对接 Casdoor 或直连 | P2 |
| G3 用户管理 | 创建/禁用/批量导入 | moredoc 后台 + 扩展 | P0 |
| G4 审计日志 | 关键操作记录 | moredoc/自研补充 | P1 |
| G5 数据备份 | 定时备份 | 运维脚本 | P0 |

### 2.4 需求裁剪说明（moredoc → 内网团队版）

moredoc 原生是面向 C 端的"百度文库"式产品，部署后需关闭以下功能：

- VIP 会员体系
- 支付功能（支付宝/微信）
- 开放注册/手机号注册（改为管理员创建账号或内部邀请）
- 积分/金币、文档售卖等运营功能
- 对外的网络爬虫采集功能

## 三、技术选型

### 3.1 选型结论

| 组件 | 选型 | 理由 |
|------|------|------|
| **知识库** | moredoc 魔豆文库 | 文件库模式契合度最高，无限级分类+批量上传+在线预览+全文检索（ES）开箱即用，存储支持本地+多云对象存储，Go 单服务资源占用低 |
| **后端语言** | Go（moredoc 自带） | 主流、性能好、单二进制部署简单 |
| **前端** | Vue（moredoc 自带 admin） | moredoc 管理端基于 Vue，二次开发延续 |
| **门户** | 轻量自研（Go Gin 或复用 moredoc 栈） | 知识库优先，门户做外壳，技术栈统一 |
| **数据库** | MySQL（moredoc 默认）或 PostgreSQL | moredoc 支持 |
| **全文检索** | Elasticsearch | moredoc v2.4.0+ 支持，中文检索效果好 |
| **对象存储** | 本地存储起步，预留 MinIO | 优先简单，按数据量增长再启用对象存储 |
| **统一认证** | 一期 moredoc 自带账号；二期 Casdoor 接钉钉 | 渐进式，先内部账号后第三方 |
| **反向代理** | Nginx | HTTPS、路由分发 |
| **部署** | Docker + Docker Compose | 统一编排，便于迁移 |

### 3.2 为什么是 moredoc

1. **定位精准匹配文件库模式。** 你的知识库需求（目录创建、知识分类、文件上传、文件描述、文件删除、文件检索、内容检索）正是 moredoc 的核心设计——它就是为"上传文件 → 分类管理 → 检索下载 → 在线预览"打造的。

2. **在线预览是关键体验。** moredoc 自动转换并在线预览 Office、PDF、EPUB 等格式，用户不下载即可阅读，这是文件库场景的核心价值。

3. **全文检索成熟。** 支持 Elasticsearch，对文档正文做全文索引，中文检索有保障。

4. **存储灵活。** 原生支持本地存储 + AWS S3 + MinIO + 阿里云 OSS 等多云对象存储，配置文件切换。当前可用本地存储起步，数据量大了再平滑切到对象存储，完全契合你"尽量支持 S3+本地"的诉求。

5. **部署轻量。** Go 单服务，资源占用低，适合单台 CPU 服务器。

6. **开源协议友好。** Apache 2.0，可自由二次开发和商用。

### 3.3 选型风险与对策

| 风险 | 对策 |
|------|------|
| moredoc 偏 toC，需裁剪改造 | 一期投入裁剪工作量，关闭运营功能 |
| 在线协作编辑能力弱 | 文件库模式以"上传文件"为主，不依赖在线协作；如需 Wiki 写作，二期可补 MrDoc |
| 二次开发需熟悉 Go+Vue | 团队若不熟，安排短期学习；moredoc 文档较完善 |
| 钉钉对接非原生 | 二期通过 Casdoor 桥接，或改造 moredoc 登录 |
| 社区规模中等 | 锁定版本，做好自有分支维护 |

### 3.4 备选方案

若实际试用后发现 moredoc 改造成本偏高，或团队明确偏好 Python：
- **MinDoc（Go）**：单二进制部署最简，适合纯技术文档，但在线预览和对象存储支持较弱。
- **MrDoc（Python/Django）**：Wiki 写作模式更强，S3 通过 django-storages 配置；适合"在线写作"而非"文件库"诉求。

## 四、实施计划

总周期约 **6~8 周**（知识库优先，门户随后）。分阶段如下。

### 阶段一：环境搭建与 moredoc 部署（第 1 周）

**目标：** moredoc 知识库在服务器上跑起来，基本可用。

任务：
- 服务器初始化（Ubuntu 22.04、Docker、Docker Compose、防火墙）
- 部署 MySQL（或 PostgreSQL）、Redis
- 拉取 moredoc 镜像，配置 `app.toml`（数据库、存储=本地、站点信息）
- 首次启动，初始化数据，验证上传/分类/预览/下载
- 配置 Nginx 反向代理 + HTTPS

**里程碑：** moredoc 可访问，能上传文件、建分类、在线预览。

### 阶段二：知识库功能配置与中文检索（第 2 周）

**目标：** 知识库核心功能就绪，检索体验达标。

任务：
- 规划并创建分类体系（技术文档/规章制度/项目资料/培训材料等）
- 部署 Elasticsearch，配置 moredoc 全文检索
- 验证中文全文检索准确性与高亮
- 测试批量上传、各格式（Office/PDF/EPUB）在线预览
- 配置文件元信息（描述、标签）规范

**里程碑：** 知识库分类清晰、上传顺畅、中文检索准确。

### 阶段三：内网化裁剪与权限/账号（第 3 周）

**目标：** 把 moredoc 改造为内网团队版。

任务：
- 关闭/隐藏 VIP、支付、积分、开放注册、文档售卖等 toC 功能
- 配置账号体系：管理员创建账号，关闭公开注册（或改邀请制）
- 批量导入初始用户（CSV）
- 配置角色与分类/文档访问权限
- 增强回收站（删除保留期与恢复，按需）
- 接入审计日志（关键操作）

**里程碑：** 系统为内网团队形态，账号与权限可控。

### 阶段四：门户外壳开发（第 4~5 周）

**目标：** 轻量门户首页 + 内容管理后台。

任务：
- 门户后端：公告 CRUD、快捷入口 CRUD、首页聚合接口（含调用 moredoc 接口取热门/最新文档）
- 门户前端：首页（公告 + 快捷入口 + 知识库入口 + 全局搜索）、公告详情页、后台管理（公告/快捷入口/站点设置）
- 门户与 moredoc 复用同一账号体系（共享登录态）
- 全局任意页面可跳转知识库

**里程碑：** 门户首页可用，公告可发布，知识库入口全局可达。

### 阶段五：集成测试与优化（第 5~6 周）

**目标：** 联调、性能与安全达标。

任务：
- 端到端测试：登录 → 门户 → 知识库 → 上传 → 检索 → 预览 → 下载
- 权限测试：不同角色访问控制
- 性能：50 并发响应时间，检索 < 1s（万级文档）
- 安全：HTTPS、上传类型白名单、XSS/CSRF、审计日志
- 备份：MySQL 每日全量 + 文件目录备份脚本，编写恢复手册

**里程碑：** 通过集成测试，性能与安全达标。

### 阶段六：试运行与上线（第 6~8 周）

**目标：** 种子用户试用后全员推广。

任务：
- 编写用户手册与管理员运维手册
- 准备初始内容（首批公告、快捷入口、示范文档）
- 10~20 人种子用户试用 1 周，收集反馈、修复问题
- 全员上线，简短培训，建立反馈渠道

**里程碑：** 系统全员可用，运行稳定。

### 里程碑总览

| 周次 | 阶段 | 里程碑 |
|------|------|--------|
| 1 | 环境+moredoc部署 | 知识库可上传/预览/下载 |
| 2 | 功能配置+中文检索 | 分类清晰、中文检索准确 |
| 3 | 内网裁剪+账号权限 | 内网团队形态、权限可控 |
| 4~5 | 门户外壳开发 | 门户首页可用、公告可发布 |
| 5~6 | 集成测试+优化 | 性能/安全达标 |
| 6~8 | 试运行+上线 | 全员可用 |

## 五、二期规划

知识库与门户稳定运行后，按优先级推进：

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 钉钉 OAuth2 登录 + 工作台跳转 | 通过 Casdoor 桥接或改造 moredoc 登录 |
| P1 | 对象存储切换 | 数据量增长后从本地存储切到 MinIO/S3 |
| P1 | 文件版本管理增强 | 文件更新的版本记录与回滚 |
| P2 | 消息通知 | 新公告/新文档的邮件或钉钉推送 |
| P2 | 数据统计 | 文档热度、用户活跃度 |
| P3 | Wiki 在线写作补充 | 如需在线协作写作，引入 MrDoc 等 |
| P3 | AI 智能检索/问答 | 接入 LLM 做知识问答 — 详细方案见 [docs/08-二期智能化方案（Adaptive多范式知识库+Agent平台）.md](08-二期智能化方案（Adaptive多范式知识库+Agent平台）.md) |
