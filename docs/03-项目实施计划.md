## 1. 技术架构设计

### 1.1 整体架构

```
                    ┌─────────────────────────────────┐
                    │           Nginx 反向代理          │
                    │     (HTTPS / 路由分发 / 静态资源)   │
                    └──────┬────────────┬──────────────┘
                           │            │
              ┌────────────▼──┐   ┌─────▼────────────┐
              │  门户前端 (React) │   │  Outline 知识库    │
              │  portal.local  │   │  wiki.local      │
              └────────┬───────┘   └──────┬───────────┘
                       │                  │
              ┌────────▼───────┐          │
              │  门户后端 (Node) │          │
              │  API Server    │          │
              └────────┬───────┘          │
                       │                  │
         ┌─────────────┼──────────────────┤
         │             │                  │
   ┌─────▼─────┐ ┌────▼─────┐  ┌────────▼────────┐
   │ PostgreSQL │ │  Redis   │  │ MinIO (S3存储)   │
   │  数据库     │ │  缓存     │  │  文件/附件存储    │
   └───────────┘ └──────────┘  └─────────────────┘
         │
   ┌─────▼──────┐
   │  Casdoor    │
   │ 统一认证网关  │
   │ (SSO/OAuth) │
   └────────────┘
```

### 1.2 技术栈明细

| 层级 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| 前端框架 | React + TypeScript | 18+ | 门户前端，与 Outline 统一 React 生态 |
| UI 组件库 | Ant Design | 5.x | 成熟的企业级 UI 组件库，中文友好 |
| 门户后端 | Node.js + Koa/Express | 20+ | 门户 API 服务，与 Outline 统一 Node.js 栈 |
| 知识库 | Outline | Latest | 开源知识库，核心功能 |
| 统一认证 | Casdoor | Latest | 开源 SSO 平台，支持自建账号+钉钉+OIDC |
| 数据库 | PostgreSQL | 15+ | Outline 和门户共用（不同 schema） |
| 缓存 | Redis | 7+ | Outline 实时协作依赖 |
| 对象存储 | MinIO | Latest | S3 兼容，文件/附件存储 |
| 反向代理 | Nginx | 1.24+ | HTTPS 终结、路由分发、静态资源 |
| 容器化 | Docker + Docker Compose | 24+ | 统一部署和管理 |
| 全文检索 | PostgreSQL pg_trgm + zhparser | — | 中文全文检索扩展 |

### 1.3 服务器资源规划

基于单台服务器部署（100~200人规模）：

| 资源 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 4核 | 8核 |
| 内存 | 8GB | 16GB |
| 系统盘 | 50GB SSD | 100GB SSD |
| 数据盘 | 100GB | 500GB（存放文件附件） |
| 操作系统 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS |

各组件内存预估：Outline ~512MB，门户后端 ~256MB，PostgreSQL ~1GB，Redis ~256MB，MinIO ~512MB，Casdoor ~256MB，Nginx ~64MB。合计约 3GB，留足余量给系统和文件缓存。

## 2. 实施阶段划分

### 阶段一：基础设施与环境搭建（第1周）

**目标：** 完成服务器环境配置，所有基础服务可运行。

**任务清单：**

1.1 服务器初始化
- 安装 Ubuntu 22.04 LTS，配置网络、防火墙
- 安装 Docker 和 Docker Compose
- 配置 SSH 密钥访问
- 创建项目目录结构

1.2 基础服务部署
- 部署 PostgreSQL，创建 Outline 数据库和门户数据库
- 部署 Redis
- 部署 MinIO，创建文件存储 Bucket
- 配置 Nginx，准备域名/IP 和 SSL 证书

1.3 编写 Docker Compose 编排文件
- 统一管理所有服务的启停
- 配置数据卷持久化
- 配置服务间网络

**交付物：** docker-compose.yml、Nginx 配置文件、环境初始化脚本、部署文档

**里程碑：** 所有基础服务启动正常，可通过浏览器访问 Nginx 默认页面

---

### 阶段二：Outline 知识库部署与配置（第2周）

**目标：** Outline 知识库可正常使用，完成基础配置。

**任务清单：**

2.1 Outline 部署
- 拉取 Outline 官方 Docker 镜像
- 配置环境变量（数据库连接、Redis、MinIO/S3、密钥等）
- 首次启动并初始化数据库 schema
- 验证 Outline 基本功能（创建文档、编辑、搜索）

2.2 Outline 功能配置
- 创建知识空间（技术文档、规章制度、项目文档、培训资料等）
- 配置默认权限策略
- 配置中文界面
- 测试文档导入导出功能

2.3 全文检索中文优化
- 安装 PostgreSQL zhparser 或 pg_jieba 扩展
- 配置中文全文检索索引
- 验证中文搜索效果

**交付物：** Outline 部署配置文档、知识空间初始化方案

**里程碑：** 团队成员可访问知识库，创建、编辑、搜索中文文档

---

### 阶段三：统一认证系统搭建（第2~3周）

**目标：** 实现自建账号登录，Outline 和门户通过 Casdoor 统一认证。

**任务清单：**

3.1 Casdoor 部署与配置
- 部署 Casdoor（Docker）
- 配置组织（Organization）和应用（Application）
- 启用用户名/密码注册登录
- 配置 OAuth2/OIDC Provider

3.2 Outline 对接 Casdoor
- 在 Casdoor 中创建 Outline 应用
- 配置 Outline 的 OIDC 环境变量，指向 Casdoor
- 测试通过 Casdoor 登录 Outline
- 验证用户信息同步（用户名、邮箱、头像）

3.3 用户管理基础配置
- 在 Casdoor 中创建管理员账号
- 批量创建初始用户（可通过 CSV 导入）
- 配置角色与权限映射

**交付物：** Casdoor 配置文档、认证集成方案文档

**里程碑：** 用户通过统一的登录页面访问 Outline，支持注册和登录

---

### 阶段四：门户后端开发（第3~4周）

**目标：** 完成门户 API 服务开发。

**任务清单：**

4.1 项目初始化
- 初始化 Node.js 项目（Koa + TypeScript）
- 配置数据库 ORM（Prisma 或 TypeORM）
- 设计门户数据库 schema（公告、快捷入口、用户扩展等）
- 配置 JWT 认证中间件（对接 Casdoor Token 验证）

4.2 公告管理 API
- 公告 CRUD 接口（创建、读取、更新、删除）
- 公告发布/撤回/置顶
- 公告分类管理
- 定时发布与自动下架逻辑

4.3 快捷入口管理 API
- 快捷入口 CRUD
- 分组管理
- 排序调整

4.4 文件管理增强 API
- 文件上传（对接 MinIO）
- 文件元信息管理（描述、标签、分类）
- 文件列表查询与筛选
- 文件下载与批量下载
- 文件删除与回收站

4.5 首页数据聚合 API
- 获取最新公告列表
- 获取热门/最近更新的知识库文档（调用 Outline API）
- 获取用户最近浏览记录

**交付物：** 门户后端代码、API 文档（Swagger/OpenAPI）

**里程碑：** 所有 API 接口开发完成，通过 Postman/curl 可正常调用

---

### 阶段五：门户前端开发（第4~6周）

**目标：** 完成门户前端界面开发。

**任务清单：**

5.1 项目初始化与通用组件
- 使用 Create React App 或 Vite 初始化项目
- 引入 Ant Design 组件库
- 开发通用布局组件（Header、Sidebar、Footer）
- 配置路由（React Router）
- 对接 Casdoor SDK 实现登录流程

5.2 门户首页
- Banner/轮播图模块
- 公告通知列表模块（展示最新公告，支持查看详情）
- 快捷入口卡片模块
- 热门知识库文档推荐模块
- 最近更新模块
- 全局搜索框（搜索公告 + 知识库文档）

5.3 公告详情页
- 公告正文展示（富文本渲染）
- 附件下载
- 相关公告推荐

5.4 知识库入口集成
- 嵌入 Outline 或提供跳转链接
- 知识库快速搜索（通过 Outline API）
- 知识库目录导航

5.5 后台管理界面
- 公告管理（列表、新建/编辑表单、富文本编辑器集成）
- 快捷入口管理
- 用户管理（列表、角色分配）
- 站点设置（Logo、名称、Banner 配置）
- 文件管理（上传、列表、删除、回收站）

**交付物：** 门户前端代码、页面截图

**里程碑：** 门户前端所有页面可正常访问和操作

---

### 阶段六：集成测试与优化（第6~7周）

**目标：** 系统整体联调，修复问题，优化性能。

**任务清单：**

6.1 功能集成测试
- 端到端测试：注册 → 登录 → 浏览门户 → 进入知识库 → 创建文档 → 搜索
- 权限测试：不同角色的访问控制是否正确
- 文件管理测试：各类型文件上传/下载/预览
- 搜索测试：中文全文检索准确性

6.2 性能优化
- 前端：代码分割、懒加载、静态资源 CDN/压缩
- 后端：数据库查询优化、索引调优
- Nginx：Gzip 压缩、缓存策略配置
- 负载测试：模拟 50 并发用户的响应时间

6.3 安全加固
- HTTPS 配置验证
- 敏感信息脱敏检查
- XSS/CSRF 防护验证
- 文件上传安全（类型白名单、大小限制）
- 审计日志功能验证

6.4 备份方案实施
- 配置 PostgreSQL 自动备份脚本（每日全量 + WAL 归档）
- MinIO 文件备份策略
- 编写数据恢复操作手册

**交付物：** 测试报告、性能测试报告、安全检查清单

**里程碑：** 系统通过集成测试，性能满足需求指标

---

### 阶段七：试运行与正式上线（第7~8周）

**目标：** 小范围试运行后全员推广。

**任务清单：**

7.1 试运行准备
- 编写用户使用手册（门户使用、知识库使用）
- 编写管理员运维手册
- 准备初始内容（首批公告、快捷入口、示范文档）
- 选定 10~20 人作为种子用户

7.2 试运行（1周）
- 种子用户试用，收集反馈
- Bug 修复和体验优化
- 监控系统稳定性和性能

7.3 正式上线
- 全员通知，提供访问地址和使用指引
- 组织简短培训（30分钟线上演示）
- 建立问题反馈渠道
- 持续监控和运维

**交付物：** 用户手册、运维手册、上线公告

**里程碑：** 系统全员可用，运行稳定

## 4. 二期规划预览

| 优先级 | 功能 | 预计工期 | 说明 |

| P2 | 评论与反馈 | 3~5天 | 知识库文档支持评论 |
| P3 | AI 智能搜索/摘要 | 1~2周 | 接入 LLM 提供智能问答 |
| P3 | 移动端 PWA | 1周 | 移动端适配优化 |

## 5. 风险评估与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| Outline 版本升级不兼容 | 中 | 高 | 锁定版本，升级前在测试环境验证 |
| 中文全文检索效果不佳 | 中 | 中 | 预留 Elasticsearch 集成方案作为 Plan B |
| 单服务器性能瓶颈 | 低 | 中 | Docker Compose 架构便于迁移到多机部署 |
| 关键人员离职 | 中 | 高 | 完善文档，避免单点依赖 |
| 用户采纳率低 | 中 | 高 | 内容先行，管理层推动，降低使用门槛 |
| Casdoor 与 Outline 集成问题 | 低 | 中 | Casdoor 官方已有 Outline 集成文档 |
