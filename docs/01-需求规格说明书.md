# 团队内部门户与知识库系统 — 需求规格说明书

> 文档版本：v1.0  
> 创建日期：2026-06-06  
> 适用范围：100~200人规模团队内部使用

---

## 1. 项目概述

### 1.1 项目背景

团队目前缺乏统一的内部信息门户和知识管理平台，日常通知公告、技术文档、规章制度、项目资料等分散在多个渠道（邮件、聊天工具、共享文件夹等），导致信息检索困难、知识流失严重、新人上手成本高。本项目旨在搭建一套集门户展示与知识库管理于一体的内部系统，统一团队信息入口，沉淀组织知识资产。

### 1.2 项目目标

- 建立统一的团队信息门户，集中展示公告、新闻、快捷入口等
- 构建结构化的知识库系统，支持文档的全生命周期管理
- 提供便捷的内容检索能力，让知识触手可及
- 支持灵活的认证体系，当前自建账号体系，未来可对接钉钉等第三方

### 1.3 用户规模与角色

| 角色 | 数量 | 职责 |
|------|------|------|
| 超级管理员 | 1~2人 | 系统配置、用户管理、全局权限 |
| 内容管理员 | 5~10人 | 门户内容编辑发布、知识库分类管理 |
| 普通用户 | 100~200人 | 浏览门户、查阅/搜索知识库、上传文档 |
| 访客 | 可选 | 仅可浏览公开内容，无需登录 |

## 2. 功能需求

### 2.1 门户 Portal 模块

#### 2.1.1 用户认证与权限

**FR-P-001 用户注册与登录**
- 支持用户名/密码方式注册和登录
- 支持邮箱找回密码
- 登录状态保持（Token/Session），支持"记住我"功能
- 预留第三方登录接口（钉钉 OAuth2、企业微信、通用 OAuth/OIDC）

**FR-P-002 角色与权限管理**
- 内置角色：超级管理员、内容管理员、普通用户、访客
- 支持自定义角色，灵活分配权限
- 权限粒度覆盖：菜单访问权限、内容编辑权限、知识库空间权限
- 支持基于组织部门的权限继承

**FR-P-003 用户管理**
- 管理员可创建、编辑、禁用、删除用户账号
- 支持批量导入用户（CSV/Excel）
- 用户个人资料管理（头像、昵称、部门、联系方式）

#### 2.1.2 门户首页

**FR-P-010 首页布局**
- 可配置的模块化首页布局，支持拖拽排序
- 预设模块包括：公告通知、新闻动态、快捷入口、热门文档、最近更新
- 支持自定义 Banner/轮播图
- 响应式设计，适配 PC 和移动端浏览器

**FR-P-011 公告通知**
- 支持发布、编辑、置顶、撤回公告
- 公告分类（通知、新闻、规章、活动等）
- 支持定时发布和到期自动下架
- 已读/未读状态跟踪
- 支持富文本编辑器编写公告内容

**FR-P-012 快捷入口**
- 管理员可配置常用系统/工具的快捷链接
- 支持图标、名称、链接地址自定义
- 分组展示（开发工具、办公系统、常用链接等）

#### 2.1.3 后台管理

**FR-P-020 内容管理后台**
- 独立的后台管理界面，仅管理员和内容管理员可访问
- 内容发布工作流：草稿 → 审核（可选） → 发布
- 支持富文本编辑器（WYSIWYG）和 Markdown 双模式
- 图片/附件上传与管理
- 内容版本历史，支持回滚

**FR-P-021 站点配置**
- 站点名称、Logo、主题色彩配置
- 首页模块启用/禁用及排序
- 页脚信息配置
- 系统维护公告设置

**FR-P-022 数据统计（可选，二期）**
- 用户活跃度统计（日/周/月活跃用户数）
- 内容阅读量统计
- 知识库热门文档排行
- 搜索关键词统计

### 2.2 知识库模块

#### 2.2.1 知识空间管理

**FR-K-001 知识空间（多空间支持）**
- 支持创建多个知识空间（如：技术文档、规章制度、项目文档、培训资料等）
- 每个空间可独立设置访问权限（公开/团队内/指定成员）
- 空间支持自定义图标和描述

**FR-K-002 目录与分类管理**
- 支持多级目录树结构（建议不超过4级）
- 拖拽方式调整目录顺序和层级
- 支持为文档添加标签（Tag），实现跨目录的分类关联
- 目录支持批量操作（移动、删除、权限设置）

#### 2.2.2 文档管理

**FR-K-010 文档创建与编辑**
- 支持在线创建文档（富文本/Markdown）
- 支持文件上传（PDF、Word、Excel、PPT、图片、压缩包等）
- 上传文件大小限制可配置（建议默认100MB）
- 文档元信息：标题、描述/摘要、作者、创建时间、最后修改时间、标签
- 文档版本管理，保留历史版本，支持版本对比和回滚

**FR-K-011 文档描述与元数据**
- 每个文档支持填写详细描述/摘要
- 自动提取并展示文档元信息（文件类型、大小、页数等）
- 支持自定义元数据字段（如：适用部门、文档等级、有效期等）

**FR-K-012 文档预览**
- 在线预览常见文档格式：PDF、图片、Markdown、纯文本
- Office 文档（Word/Excel/PPT）在线预览（可集成 OnlyOffice 或 LibreOffice）
- 代码文件语法高亮预览

**FR-K-013 文档下载与导出**
- 支持单文件下载
- 支持目录批量打包下载
- 在线文档支持导出为 PDF/Markdown/HTML 格式

**FR-K-014 文档删除与回收站**
- 文档删除需二次确认
- 删除后进入回收站，保留30天（可配置）
- 回收站支持恢复和彻底删除
- 支持管理员批量清理回收站

#### 2.2.3 检索功能

**FR-K-020 文件检索（元数据检索）**
- 按文件名、标签、分类目录、上传者、时间范围等条件筛选
- 支持组合条件检索
- 检索结果支持排序（相关度、时间、名称）

**FR-K-021 内容检索（全文检索）**
- 对文档正文内容进行全文索引
- 支持中文分词（jieba 或 IK Analyzer 等）
- 对上传的 PDF、Word 文档提取文本后纳入全文索引
- 搜索结果高亮显示匹配关键词
- 搜索建议/自动补全（可选，二期）

**FR-K-022 全局搜索入口**
- 在门户和知识库的任何页面都可访问搜索功能
- 支持快捷键呼出搜索框（如 Ctrl+K）
- 搜索范围支持切换：全站 / 当前空间 / 当前目录

#### 2.2.4 协作功能（二期可选）

**FR-K-030 评论与反馈**
- 文档页面支持评论功能
- 评论支持 @提及用户
- 文档点赞/收藏功能

**FR-K-031 文档关联**
- 文档间可建立关联关系
- 自动展示相关文档推荐

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | 首页 < 2秒，文档页 < 3秒 |
| 并发用户数 | 支持50人同时在线操作 |
| 搜索响应时间 | 全文检索 < 1秒（万级文档规模） |
| 文件上传速度 | 受限于网络带宽，100MB文件 < 30秒（内网） |
| 数据库容量 | 支持10万+文档记录 |

### 3.2 安全要求

- 所有通信使用 HTTPS 加密
- 密码存储使用 bcrypt 等单向哈希算法
- 防止常见 Web 攻击：XSS、CSRF、SQL注入
- 敏感操作记录审计日志（登录、删除、权限变更等）
- 文件上传安全检查（文件类型白名单、病毒扫描接口预留）
- Session/Token 超时自动登出

### 3.3 可用性要求

- 系统可用率 > 99%（单机部署场景）
- 数据备份：每日自动全量备份，保留最近30天
- 系统异常时显示友好错误页面
- 支持平滑重启，不中断在线用户操作

### 3.4 可维护性要求

- 提供系统运行状态监控页面
- 日志分级输出（DEBUG/INFO/WARN/ERROR）
- 配置外部化，支持环境变量或配置文件修改
- 支持 Docker 容器化部署，便于迁移和扩展
- 提供数据导入/导出工具

### 3.5 兼容性要求

- 浏览器：Chrome（最新2个版本）、Firefox、Edge、Safari
- 移动端：通过响应式设计适配手机和平板浏览器
- 无需开发原生 App（一期），PWA 作为二期可选

## 4. 系统集成需求

### 4.1 认证集成

| 阶段 | 认证方式 | 说明 |
|------|----------|------|
| 一期 | 自建账号体系 | 用户名密码注册登录，管理员可批量创建 |
| 二期 | 钉钉 OAuth2 | 从钉钉工作台一键跳转登录，自动同步用户信息 |
| 可选 | LDAP/AD | 对接企业目录服务 |
| 可选 | 通用 OAuth2/OIDC | 支持标准协议的第三方身份认证 |

### 4.2 存储集成

- 文件存储：本地磁盘存储（一期），可扩展为 MinIO/S3 兼容对象存储
- 数据库：PostgreSQL（推荐）或 MySQL

### 4.3 通知集成（二期可选）

- 站内消息通知
- 邮件通知（SMTP）
- 钉钉消息推送（Webhook/API）

## 5. 数据模型概要

### 5.1 核心实体关系

```
用户(User) ──── 角色(Role) ──── 权限(Permission)
   │
   ├── 发布 ──── 门户内容(PortalContent)
   │                ├── 公告(Announcement)
   │                ├── 新闻(News)
   │                └── 快捷入口(QuickLink)
   │
   └── 管理 ──── 知识库(KnowledgeBase)
                    ├── 知识空间(Space)
                    │      └── 目录(Category)
                    │             └── 文档(Document)
                    │                    ├── 文件附件(Attachment)
                    │                    ├── 版本记录(Version)
                    │                    └── 标签(Tag)
                    └── 全文索引(FullTextIndex)
```

### 5.2 关键数据表

| 数据表 | 说明 | 关键字段 |
|--------|------|----------|
| users | 用户信息 | id, username, email, password_hash, role_id, department, status |
| roles | 角色定义 | id, name, permissions(JSON), description |
| portal_contents | 门户内容 | id, title, content, type, status, author_id, publish_time, expire_time |
| spaces | 知识空间 | id, name, description, icon, visibility, owner_id |
| categories | 目录分类 | id, name, parent_id, space_id, sort_order |
| documents | 文档 | id, title, description, content, category_id, author_id, file_path, file_type, file_size, version |
| document_versions | 版本历史 | id, document_id, version_num, content, changed_by, change_note |
| tags | 标签 | id, name |
| document_tags | 文档标签关联 | document_id, tag_id |
| attachments | 附件 | id, document_id, file_name, file_path, file_size, mime_type |
| audit_logs | 审计日志 | id, user_id, action, target_type, target_id, detail, created_at |

## 6. 界面原型要求

### 6.1 门户首页

- 顶部：系统 Logo + 名称 + 全局搜索框 + 用户头像/菜单
- 主体区域：轮播 Banner → 公告通知列表 → 快捷入口卡片 → 热门文档推荐
- 侧边栏（可选）：知识库目录导航、最近浏览
- 底部：版权信息、系统版本

### 6.2 知识库页面

- 左侧：空间列表 + 目录树导航
- 中间：文档内容区（阅读/编辑模式切换）
- 右侧（可选）：文档元信息面板（作者、时间、标签、版本历史）
- 顶部：面包屑导航 + 操作按钮（编辑、下载、删除、分享）

### 6.3 后台管理

- 左侧：管理菜单（用户管理、角色权限、门户内容、知识库管理、系统设置、审计日志）
- 右侧：管理操作区域
- 管理界面风格统一，采用表格+表单的经典管理后台布局

## 7. 约束与假设

### 7.1 约束条件

- 部署在单台 CPU 服务器上，需关注资源优化
- 一期不考虑高可用和集群部署
- 优先使用成熟的开源项目作为基础，避免从零开发
- 技术栈选择主流方案，确保社区支持和长期维护性

### 7.2 假设条件

- 服务器操作系统为 Linux（推荐 Ubuntu 22.04/CentOS 7+）
- 服务器配置不低于：4核 CPU、8GB 内存、100GB 磁盘
- 已具备域名和 SSL 证书（内网可用自签名证书）
- 团队内有1~2名具备基础运维能力的工程师
